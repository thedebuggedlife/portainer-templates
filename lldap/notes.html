Application Data

The container is pre-configured to store persistent data for LLDAP under /srv/appdata/lldap.

You may change the location by expanding Show advanced options below, then look under Volume mapping.

To create the parent application data folder:

sudo mkdir -p /srv/appdata/lldap
# Replace with the PID for LLDAP (the default is 1000)
LLDAP_UID=1000
sudo chown $LLDAP_UID /srv/appdata/lldap

Secret Files

The default configuration points LLDAP to a pair of secret files for JWT secret and KEY seed. This is the recommended way to provide these values to LLDAP instead of environment variables in clear text. The default location of the files is /srv/appdata/lldap/jwt.secret and /srv/appdata/lldap/key.seed respectively.

To generate these files you can run the following:

wget -O- https://raw.githubusercontent.com/thedebuggedlife/portainer-templates/refs/heads/main/lldap/generate-secrets.sh | bash

Base Domain Name

Use the following format: dc=example,dc=com if your domain is example.com
